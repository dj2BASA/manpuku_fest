●概要

「fes_result.html」が画面描画用のファイルになります。
「fest_result_output_settings.exe」が設定ツールです。
基本的にはこの2つのどちらかを起動させます。


●fes_result.html について

・OBSで「ブラウザ」として「fes_result.html」を取り込むか、
  Chrome等で開くことを想定しています。

・OBSで「ブラウザ」として取り込んだときは設定ツールで設定した情報が適用されます。
  Chrome等で直接開いた時は設定が読み込まれず、初期値で動きます。設定ファイルが読み込めない旨のメッセージは正しい動作です。

・OBSでブラウザとしてソースを追加できたら、「対話 (操作)」ボタンを押すと操作できるようになります。
  ウインドウをクリックすることで先に進みます。
  Chrome等で開いた場合はそのままウインドウをクリックすると先に進みます。

●fest_result_output_settings.exe について

・データの設定をすることができるツールです。Windows10にて動作を確認しています。

・フォーム名が「Form1」だったり、リストの列名が「ColumnHeader」だったり、矢印ボタンが動作しなかったり、
  まだ動作は最低限です。故に想定外な動作には弱いです。
  「setting_data」フォルダの素材やフォルダそのものが無かったりすると恐らくエラーで動かないです。

・出力ボタンで「settings.xml」と「settings.json」と「最終結果用の得点画像データ」を出力します。
現状「settings.json」は使っていませんが後々はこっちを読み込むようにしたい…

●もろもろの.jsファイルについて

・描画のためのファイルです。
  ある程度は整備して整備前よりはまだだいぶマシになりました。
  ですがまだ現状かなりあっちこっちな書き方なので複雑スパゲッティ化してます。

・ひっそり。
  
  debug.js の上の方にある
  var debug_is_display = false;
  を false から true にすると、デバッグ用の出力を表示します。
  
  sound.js の上の方にある
  var is_all_mute = false;
  を false から true にすると、音声がミュートされ、再生されなくなります。


●もろもろの.dllについて

・設定ツールを動かすために必要なやつです。ツール作成時に自動で出力されます。無いと動きません。
